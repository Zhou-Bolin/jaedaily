面试题 第一题

约束预算模型路由

▶ YouTube ▶ B站↗ 原始论文
题目 92/100

你负责一个线上 AI 助手的模型路由系统，每天 500 万请求，可选择 Small/Medium/Large 三个模型，也允许级联调用。Large 质量最高但成本和延迟最高；Small 成本仅为 Large 的 8%，但在复杂推理、长上下文、多语言请求上失败率高。产品约束是：整体质量相比全量 Large 下降不超过 1%，平均成本不超过全量 Large 的 35%，端到端 p95 延迟不超过 900ms；日志中只有旧路由策略产生的样本，真实质量标签稀疏且延迟 1-3 天返回，线上还存在节假日和新功能导致的分布漂移。

请设计一个可落地的模型路由算法：包括如何定义优化目标，如何训练路由器和评估离线收益，线上如何做探索、校准、漂移处理和回退；同时分析推理复杂度、训练复杂度，以及你认为最容易踩坑的边界条件。

参考回答

我会把问题建模成带约束的上下文决策问题。对每个请求特征 x，包括 prompt 长度、语言、业务场景、用户等级、检索命中率、历史失败率、预估输出长度、风险标签等，动作 a 可以是直接调用 S/M/L，也可以是 S→Verifier→M/L 这样的级联策略。目标不是单纯预测哪个模型最好，而是在质量、成本和尾延迟之间做约束优化，例如最大化 E[Q(a,x)]，约束 E[C(a,x)]≤0.35C_L，P95(T(a,x))≤900ms，且 E[Q(L,x)-Q(a,x)]≤1%。

核心算法我会分三层。第一层是离线质量/收益建模：用历史日志、人审、用户反馈、自动评测器构造质量标签，训练每个模型在请求 x 上的质量预测器 q_a(x)、成本预测器 c_a(x)、延迟分位数预测器 t_a,p95(x)，以及“升级收益” Δq_{S→M/L}(x)。因为日志来自旧策略，有选择偏差，需要用 propensity score 做 IPS 或 doubly robust 估计；如果旧系统完全没有探索，则必须补一批分层随机流量，否则不能可靠估计反事实收益。第二层是约束路由：用拉格朗日形式 score(a,x)=q_a(x)-λc_a(x)-μr_a(x)，其中 r_a(x) 表示尾延迟风险或 SLA 违约概率；λ、μ 由离线网格搜索或线上 dual update 调节。实际线上可简化成“选择满足质量置信下界的最便宜动作”，例如若 Small 的校准通过概率高于阈值且风险低，则走 Small；否则走 Medium；高风险、OOD、强安全场景直接 Large。第三层是级联：先用 Small 生成草稿，再用轻量 verifier 判断是否足够好，只有 verifier 低置信或检测到事实性/安全/格式风险时升级到 M/L。级联要注意总延迟，不能让 S+Verifier+L 的 p95 超过直接 L。

复杂度上，线上路由器若用 GBDT/小型 MLP，对 K 个候选动作打分复杂度约 O(Kd) 或 O(K·model_size)，K 很小，通常远低于一次 LLM 推理；级联 verifier 若是小模型，需要计入额外几十毫秒延迟。离线训练若有 N 条样本、K 个动作，直接训练每动作一个质量模型是 O(NK) 级别，若使用共享表示则近似 O(N)。在线统计需要维护按业务桶、模型、时间窗口的成本、质量、p95、超时率，空间约 O(BK)。

工程上我会做几件事：第一，质量预测必须校准，可用 isotonic/temperature calibration 或 conformal prediction 给出置信区间，低置信一律升级；第二，p95 不能用平均延迟替代，要按输出 token 长度、并发队列、地区、模型冷启动分别建模；第三，线上做小流量分层探索，例如 1%-3% ε-greedy 或 Thompson sampling，并设置每业务线预算保护；第四，用 shadow evaluation 和回放评估新路由器，指标同时看质量差、成本、p95、p99、升级率、投诉率；第五，漂移检测包括输入分布 PSI/KL、路由分布变化、verifier 置信度坍缩、分桶质量下降，一旦异常自动提高 λ/μ 或回退到保守策略；第六，高安全、付费关键路径、长上下文截断风险、工具调用失败等场景要有硬规则兜底。

回答分析

强回答应该先把“模型路由”形式化为约束优化或上下文 bandit，而不是简单分类问题；要明确质量、成本、p95 延迟三者的目标关系，并意识到旧策略日志存在反事实评估偏差。好的方案会包含质量预测、成本/延迟预测、校准、级联 verifier、Lagrangian 或阈值策略、线上探索、漂移监控和回退机制。复杂度分析需要覆盖路由器推理开销、级联额外延迟、离线训练规模，以及按分桶维护统计的成本。常见错误包括：只按 prompt 长度路由；只优化平均延迟不看尾延迟；直接用旧日志监督学习而不处理选择偏差；把 LLM-as-judge 当绝对真值；没有探索流量导致无法估计 Medium/Large 的反事实收益；级联策略省了钱但把 p95 打爆。面试官会重点追问候选人如何证明质量只下降 1%、如何处理标签延迟、以及当线上分布突然漂移时如何自动止损。

代码题 第二题

匹配模式数组的子数组数目 II

▶ YouTube ▶ B站↗ LeetCode
题目 95/100

给你一个下标从 0 开始长度为 `n` 的整数数组 `nums` ，和一个下标从 `0` 开始长度为 `m` 的整数数组 `pattern` ，`pattern` 数组只包含整数 `-1` ，`0` 和 `1` 。

大小为 `m + 1` 的子数组 `nums[i..j]` 如果对于每个元素 `pattern[k]` 都满足以下条件，那么我们说这个子数组匹配模式数组 `pattern` ：

  • 如果 `pattern[k] == 1` ，那么 `nums[i + k + 1] > nums[i + k]`
  • 如果 `pattern[k] == 0` ，那么 `nums[i + k + 1] == nums[i + k]`
  • 如果 `pattern[k] == -1` ，那么 `nums[i + k + 1]

请你返回匹配 `pattern` 的 `nums` 子数组的 数目 。

示例 1：

输入：nums = [1,2,3,4,5,6], pattern = [1,1] 输出：4 解释：模式 [1,1] 说明我们要找的子数组是长度为 3 且严格上升的。在数组 nums 中，子数组 [1,2,3] ，[2,3,4] ，[3,4,5] 和 [4,5,6] 都匹配这个模式。 所以 nums 中总共有 4 个子数组匹配这个模式。

示例 2：

输入：nums = [1,4,4,1,3,5,5,3], pattern = [1,0,-1] 输出：2 解释：这里，模式数组 [1,0,-1] 说明我们需要找的子数组中，第一个元素小于第二个元素，第二个元素等于第三个元素，第三个元素大于第四个元素。在 nums 中，子数组 [1,4,4,1] 和 [3,5,5,3] 都匹配这个模式。 所以 nums 中总共有 2 个子数组匹配这个模式。

提示：

  • `2 6`
  • `1 9`
  • `1
参考解法

参考解法：把 nums 的相邻大小关系转换成一个由 -1、0、1 组成的序列，然后问题就变成：统计 pattern 在这个关系序列中出现了多少次。使用 KMP 可以在线性时间内完成匹配，并且没有哈希冲突风险。

python
from typing import List


class Solution:
    def countMatchingSubarrays(self, nums: List[int], pattern: List[int]) -> int:
        n = len(nums)
        m = len(pattern)

        if n < m + 1:
            return 0

        # 1. 构建 pattern 的 KMP 前缀函数 pi
        # pi[i] 表示 pattern[0..i] 的最长相等真前后缀长度
        pi = [0] * m
        j = 0
        for i in range(1, m):
            while j > 0 and pattern[i] != pattern[j]:
                j = pi[j - 1]
            if pattern[i] == pattern[j]:
                j += 1
            pi[i] = j

        # 2. 在线扫描 nums 的相邻比较结果，不必额外构造长度 n - 1 的数组
        ans = 0
        j = 0  # 当前已经匹配的 pattern 长度

        for i in range(n - 1):
            if nums[i + 1] > nums[i]:
                cur = 1
            elif nums[i + 1] == nums[i]:
                cur = 0
            else:
                cur = -1

            while j > 0 and cur != pattern[j]:
                j = pi[j - 1]

            if cur == pattern[j]:
                j += 1

            # pattern 完整匹配一次
            # 对应一个长度为 m + 1 的 nums 子数组
            if j == m:
                ans += 1
                # 继续寻找可重叠匹配
                j = pi[j - 1]

        return ans
解题分析

关键思想：

1. 将子数组匹配条件转化为相邻关系匹配 对 nums 中每一对相邻元素 nums[i] 和 nums[i + 1]，生成一个关系值： - nums[i + 1] > nums[i]，记为 1 - nums[i + 1] == nums[i]，记为 0 - nums[i + 1] < nums[i]，记为 -1

这样 nums 会被转换成一个长度为 n - 1 的关系数组。题目要求统计长度为 m + 1 的 nums 子数组，本质就是统计 pattern 在这个关系数组中出现了多少次。

2. 为什么使用 KMP n 最大可达 10^6，朴素枚举每个起点再检查 m 个位置，最坏复杂度是 O(nm)，会超时。 KMP 可以在 O(n + m) 时间内完成模式串匹配，并且相比 Rolling Hash 不存在哈希冲突问题，更适合面试现场作为主解法。

3. 在线扫描优化空间 不一定要显式构造关系数组 diff。遍历 nums 时直接计算当前相邻关系 cur，然后将 cur 喂给 KMP 状态机即可。 这样额外空间只需要 O(m)，用于存储 pattern 的前缀函数。

复杂度分析：

- 时间复杂度：O(n + m) 构建 KMP 前缀函数需要 O(m)，扫描 nums 的相邻关系需要 O(n)。

- 空间复杂度：O(m) 只存储 pattern 的前缀函数 pi。如果先构造 diff 数组，则空间会变成 O(n + m)。

边界条件：

  • pattern 长度为 1：只需要统计 nums 中相邻元素满足某一种关系的次数，KMP 仍然适用。
  • nums 中存在大量相等元素：比较结果为 0，需要正确处理。
  • 匹配可以重叠：例如关系序列 [1,1,1] 中 pattern [1,1] 出现 2 次，匹配后必须令 j = pi[j - 1]，不能直接清零。
  • m + 1 不能超过 n；虽然题目保证 m < n，但工程实现中可以加保护。

常见错误：

  • 把 pattern[k] == 1 理解反了。它要求 nums[i + k + 1] > nums[i + k]。
  • 忘记匹配的是长度 m + 1 的 nums 子数组，而不是长度 m 的子数组。
  • KMP 完整匹配后直接 j = 0，导致漏掉重叠匹配。
  • 使用 Rolling Hash 但没有处理哈希冲突风险；如果使用单哈希，理论上可能误判。
  • 构造 diff 后枚举比较 pattern，复杂度仍可能退化为 O(nm)。

面试评价重点：

高质量答案应能主动说明“子数组条件”与“相邻关系数组匹配”之间的一一对应关系，并选择 KMP 做线性匹配。代码中需要正确处理前缀函数、失配回退、完整匹配后的重叠匹配，以及 -1、0、1 三种关系。
