面试题 第一题

流式多模态记忆检索

▶ YouTube ▶ B站↗ 原始论文
题目 91/100

你要设计一个第一视角 AI 记忆系统：用户每天佩戴设备 8 小时，输入为 720p/30fps 视频、16kHz 音频和少量传感器元数据。设备端可用内存 4GB、长期存储每天最多 16GB、平均功耗受限，要求在线写入不阻塞采集；用户可用文本/语音/图片查询“我早上把钥匙放哪了”“刚才会议里谁提到预算”，查询 p95 延迟 < 800ms，且大部分数据必须本地处理。 请给出你的端到端算法方案：如何做流式事件切分、多模态表征、索引与检索、记忆压缩/淘汰、离线合并与模型版本演进。要求说明复杂度、关键阈值如何自适应、如何处理误切分、数据漂移、隐私与边界场景。

参考回答

我会把问题建模为带预算约束的在线记忆构建与检索：对连续流 x_t 构建有层级的记忆 M，使得在存储 B、写入延迟 L_w、查询延迟 L_q 约束下最大化事件级检索召回和答案可信度。核心不是保存所有帧，而是保存“近期原始片段 + 事件级摘要 + 多模态向量 + 时间索引”。

写入侧分四层。第一层是低成本采样与缓存：视频按场景变化自适应采样，例如 1-3fps 基础采样，检测到运动、OCR、人物、物体变化时升采样；音频用 VAD 切语音段，非语音段只保留低维声学指纹。最近 5-15 分钟保留环形原始缓存，便于误切分后回溯。第二层是在线事件切分：对每个时间窗提取轻量视觉 embedding、ASR/OCR 文本 embedding、音频 embedding 和位置/时间特征，维护滑动窗口统计量。当相邻窗口的融合 embedding 距离、场景分类变化、VAD/说话人变化超过自适应阈值时产生候选边界。阈值不固定，而是用最近一段时间的距离分布分位数或 EWMA 均值方差校准，例如 d_t > μ_t + kσ_t，同时设置最短/最长事件长度，避免抖动。边界先软提交，后台可根据后续上下文做 merge/split 修正。

第三层是多模态记忆表示。每个事件存：时间范围、少量关键帧、ASR 片段、OCR、检测到的人/物/地点、全局融合向量、分模态向量、置信度和 provenance。融合向量用于粗召回，分模态向量用于避免文本淹没视觉证据。向量可做 int8 或 PQ 压缩；文本摘要用小模型增量生成，必要时只保存结构化 facts。第四层是索引：维护时间 B-tree/interval index、向量 ANN 索引如 HNSW 或 IVF-PQ、实体倒排索引和传感器过滤索引。查询时先解析意图和约束，例如“早上”“钥匙”“谁提到预算”，转为时间过滤、实体过滤和向量查询；然后多路召回 topK 事件，做 temporal expansion，把命中事件前后若干分钟加入候选；最后用更强的跨模态 reranker 或小型 VLM/LLM 对证据片段重排并生成答案，答案必须带时间和证据来源。

记忆压缩采用层级保留策略。近期数据保留原始片段，中期保留关键帧和语音片段，长期只保留摘要、实体、embedding 和少量高价值证据。事件价值分数可由 recency、salience、diversity、用户访问频次、置信度和隐私等级组成，低价值相似事件聚类合并。为了避免只保留“显著事件”导致日常小物体查询失败，对常见物体位置类事件单独维护轻量状态表，例如 object -> last seen locations with confidence。淘汰时优先丢原始数据，再降采样关键帧，最后才删结构化记忆。

复杂度上，若每天采样后有 T 个窗口、N 个事件、向量维度 d，写入主要是编码成本 O(T·C_encoder)，事件边界判断 O(Td)，ANN 插入约 O(log N) 或摊销近似常数到对数级。存储约为 O(N·d_q + N·metadata + keyframes)，其中 d_q 是压缩后的向量长度。查询阶段时间过滤 O(log N + m)，ANN 召回近似 O(log N) 到 O(sqrt N)，rerank 为 O(K·C_rerank)，K 通常控制在 50-200，保证 p95 < 800ms。工程上会把编码异步流水线化，优先保证采集不丢帧；低电量时降采样、暂停重模型摘要，只写入轻量索引，充电时后台补全。

风险点主要有五类。第一，误切分会影响召回，所以事件边界必须允许重叠窗口和后台修正，检索也不能只搜单个事件，要做时间扩展。第二，模型升级导致 embedding 不兼容，需要索引版本化、双写一段时间、后台重建或用 adapter 对齐旧向量。第三，数据漂移，例如用户换环境或摄像头角度变化，需要监控 embedding 分布、边界触发率、召回点击率，并动态调整阈值。第四，隐私上默认本地处理，敏感区域、人脸、屏幕内容可加密或只存摘要，支持用户按时间/地点删除并同步删除索引。第五，评估不能只看向量 recall，要构造事件级问答集，分别评估 time localization、evidence precision、answer exactness、存储占用、功耗和端侧 p95/p99 延迟。

回答分析

强回答应先把问题抽象成预算约束下的流式记忆构建，而不是直接堆一个大模型；应覆盖在线事件切分、多模态特征、层级索引、查询重排、记忆压缩、模型升级和隐私治理。算法上要说明为什么需要时间索引、向量索引、实体倒排联合使用，以及为什么检索要做 temporal expansion 和 evidence-based rerank。复杂度上至少要能估算写入、索引、查询和存储随事件数增长的关系。工程上要体现端侧功耗、异步流水、降级策略、原始缓存窗口、索引版本化和后台补偿。常见错误是只说“用 CLIP+向量数据库+RAG”，没有事件边界、淘汰策略和 p95 延迟设计；或者保存所有视频，忽略存储预算；或者只做文本 ASR，无法回答视觉位置类问题。面试官会重点追问阈值如何自适应、误切分如何恢复、embedding 升级如何保持线上一致、以及如何证明系统真的“记住了”而不是靠语言模型幻觉。

代码题 第二题

T 秒后青蛙的位置

▶ YouTube ▶ B站↗ LeetCode
题目 95/100

给你一棵由 `n` 个顶点组成的无向树，顶点编号从 `1` 到 `n`。青蛙从 顶点 1 开始起跳。规则如下：

  • 在一秒内，青蛙从它所在的当前顶点跳到另一个 未访问 过的顶点（如果它们直接相连）。
  • 青蛙无法跳回已经访问过的顶点。
  • 如果青蛙可以跳到多个不同顶点，那么它跳到其中任意一个顶点上的机率都相同。
  • 如果青蛙不能跳到任何未访问过的顶点上，那么它每次跳跃都会停留在原地。

无向树的边用数组 `edges` 描述，其中 `edges[i] = [ai, bi]` 意味着存在一条直接连通 `ai` 和 `bi` 两个顶点的边。

返回青蛙在 `t` 秒后位于目标顶点 `target` 上的概率。与实际答案相差不超过 `10-5` 的结果将被视为正确答案。

示例 1：

输入：n = 7, edges = [[1,2],[1,3],[1,7],[2,4],[2,6],[3,5]], t = 2, target = 4 输出：0.16666666666666666 解释：上图显示了青蛙的跳跃路径。青蛙从顶点 1 起跳，第 1 秒 有 1/3 的概率跳到顶点 2 ，然后第 2 秒 有 1/2 的概率跳到顶点 4，因此青蛙在 2 秒后位于顶点 4 的概率是 1/3 * 1/2 = 1/6 = 0.16666666666666666 。

示例 2：

输入：n = 7, edges = [[1,2],[1,3],[1,7],[2,4],[2,6],[3,5]], t = 1, target = 7 输出：0.3333333333333333 解释：上图显示了青蛙的跳跃路径。青蛙从顶点 1 起跳，有 1/3 = 0.3333333333333333 的概率能够 1 秒 后跳到顶点 7 。

参考解法

参考解法如下，采用 DFS 在树上模拟概率转移： from typing import List

class Solution: def frogPosition(self, n: int, edges: List[List[int]], t: int, target: int) -> float: graph = [[] for _ in range(n + 1)]

for a, b in edges: graph[a].append(b) graph[b].append(a)

def dfs(node: int, parent: int, elapsed: int, prob: float) -> float: # 已经经过 t 秒，只看当前是否正好在 target if elapsed == t: return prob if node == target else 0.0

# 树上从 1 出发，不能回到父节点，因此未访问邻居就是除 parent 外的邻居 children = [nxt for nxt in graph[node] if nxt != parent]

# 如果提前到达 target： # 1. target 没有未访问子节点，青蛙会一直停在这里，概率仍然有效 # 2. target 还有未访问子节点，时间未结束时青蛙必须继续跳走，最终不在 target if node == target: return prob if not children else 0.0

# 当前不是 target，且已经无路可走，只会停在当前节点，无法再到达 target if not children: return 0.0

next_prob = prob / len(children)

for child in children: ans = dfs(child, node, elapsed + 1, next_prob) if ans > 0: return ans

return 0.0

return dfs(1, 0, 0, 1.0)

解题分析

关键思想：因为输入是一棵树，从根节点 1 出发后，青蛙不能回到访问过的点，所以每个节点的“可跳节点”就是它在以 1 为根时的子节点。DFS 过程中维护当前节点、父节点、已经用掉的时间以及到达当前节点的概率。每次从当前节点跳向所有未访问子节点时，概率平均分配，即除以可选子节点数量。最重要的判断是提前到达 target 的情况：如果 elapsed < t 且 target 还有未访问子节点，青蛙下一秒一定会离开，因此 t 秒后在 target 的概率是 0；如果 target 没有未访问子节点，青蛙会一直停留在 target，因此概率保持不变。复杂度：建图 O(n)，DFS 最多访问每个节点一次，时间复杂度 O(n)，空间复杂度 O(n)，递归深度 O(n)。边界条件：n = 1 且 target = 1 时应返回 1；target 是叶子且提前到达时可以停留；target 非叶子且提前到达时概率为 0；t 小于到达 target 的路径长度时概率为 0；t 正好等于路径长度时返回路径概率。常见错误：把度数直接当作分母，忘记排除父节点；提前到达 target 后直接返回概率，忽略 target 还有子节点会继续跳走；叶子节点停留规则处理错误；用普通图 DFS 时没有避免回到父节点。面试追问通常会关注：为什么只用 parent 就能表示 visited、如何推导 target 提前到达时的条件、能否用 BFS 或只沿唯一根到 target 路径计算概率。
